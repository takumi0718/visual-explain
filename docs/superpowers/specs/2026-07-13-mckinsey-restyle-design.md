# McKinsey 流ビジュアル標準 v3 — canonical 12 形式リスタイル設計

日付: 2026-07-13
状態: ユーザーレビュー待ち
モック（見た目の正）: `.visual-explain/2026-07-13-mckinsey-restyle-candidates-mock.html`
参照素材: `McKinsey/` フォルダ 8 枚＋ユーザー添付 3 枚（waterfall・enumeration・chevron）

## 背景と目的

4 レンズレビュー（2026-07-13）で、canonical 図形式は「UI 洗練 4.5/10・図の平均 4.4/10」と判定され、pyramid・waterfall の出荷不可バグ、`--radius` 未定義、ダークモードの面塗り欠陥、確度語彙の分裂が特定された。ユーザーは制約体系の目的を「異種 LLM 間の品質フロア保証」と再定義し、目的を超過する制約の緩和と、マッキンゼー/BCG 流スライドの見た目の再現を指示した。

本設計は **図解コンポーネントの視覚層の全面リスタイル**である。品質フロア（固定領域バイト一致・四層検証・fail-closed・確度/出典規律・外部依存ゼロ）は一切変更しない。P1（ナビゲーション）・P2（信頼済み双方向部品・line チャート・data: 画像）は別 spec で扱う。

## 決定事項（ユーザー確認済み）

1. **配色 = 単一固定パレット「紺」**。資料・モデルを問わず同一の見た目（品質フロアと両立）。
2. **説明文中の色付き太字強調を採用、1 説明文 1 フレーズ上限**（checker 強制）。
3. **waterfall は renderer-SVG へ移行**（CSS バー版は廃止、併存させない）。
4. **対象 = canonical 10 全形式＋bars・kpi の canonical 昇格**（計 12 形式）。
5. 実装アプローチ = **全形式 `@2` への原子的バージョンアップ**（拡張ゲート 10 手順に準拠、部分適用や新旧並行を作らない）。

## 紙面規約（全形式共通）

- 図形は**角丸なし・影なし**。区切りは塗り面と細罫のみ。
- 定量図（waterfall / slope / bars）は**図ヘッダ必須**: 図タイトル（太字）＋`単位: X`（グレー小）＋細罫。IR に `unitLabel` を必須フィールドとして追加する。
- 図中の文字は本文より**太く・大きく**（コンセプト文字は 1.0625rem/700 目安）。行間・図間の余白は大きく取る。
- コンセプト箱＋右説明の分離構造（既存の enumeration/chevron 説明分離仕様）は維持する。

## デザイントークン（骨格に追加する閉じた集合）

骨格 `:root` に図解専用トークンを追加する。値はモックの実測どおり。

## 記法の追加・変更

- **文中強調**: `<strong class="dg-em">`（描画は `--dg-emphasis`・700）。1 つの description につき最大 1 個。checker で超過を FAIL。
- **負値の ▲ 表記**: waterfall の負の delta は `▲{絶対値}` で表示する。`-` 表記は使わない。
- **ハイライト 1 箇所ルール**: `--dg-highlight`（teal）と matrix の淡強調セルは 1 図につき 1 箇所まで。checker で強制。
- **確度語彙の統一**: 図中チップの「確定/推定」を廃止し、design-system の 3 語「確認済み/推論/未確認」に統一。checker 第 3 層で語彙を固定。
- 単位・数値の常時併記（棒だけで値を伝えない）は現行規約のまま。

## コンポーネント別仕様（12 形式）

見た目の正はモック①〜⑫。ここでは構造上の要点だけ列挙する。

1. **enumeration@2** — 左: 紺コンセプト箱（13rem・中央揃え・白抜き太字）、右: `・` 始まりの説明（文中強調可）。行間は広く（--space-4）。番号は箱内に含めてよい。
2. **chevron@2** — 紺の下向き五角形（clip-path、先端 1.9rem）。`loop:true` はグレー細レール（2.5px）の**途切れない単一経路**: 最終段下端の頂点 → 縦降下 → 下辺を左へ → 左辺を上昇 → 上辺を右へ → 先頭段上辺の中央へ縦降下 → **矢尻が上辺に接触**。レール中心線の x はコンセプト列の中心（`--chv-cx`）で共有し、レールは見出し・本文と重ならないよう wrap の padding で領域を確保する。
3. **matrix@2** — 概念比較型: 紺の列ヘッダ箱＋グレー行ラベル箱＋紺 1.5px アウトラインの白抜き値セル（中央・太字）。淡強調セルは 1 箇所まで。文字量の多い密な表は現行の細横罫スタイルに紺見出しを適用した表記へ写像する（縦罫なしは維持）。
4. **flow@2** — 紺ノード（中間状態はアウトライン白抜き）、遷移は下向き三角矢印＋**矢印直近の遷移ラベル**。エッジラベルを右端へ離す現行 grid を廃止。
5. **pyramid@2** — 紺 3 階調（上=濃）。tiers コンテナは確定幅 `min(100%, 24rem)`、各 tier は `min-width: max-content`＋`white-space: nowrap` で 1 文字折返しを構造的に禁止。最大 4 層は tier 4 に `--dg-neutral` を割当。
6. **stairs@2** — 上昇階段。到達済み=紺、現在地=teal＋`← 現在地`、未到達=`--dg-neutral`。
7. **logic-tree@2** — 紺 root＋第 2 階調の枝ヘッダ＋アウトライン leaf を**左→右のエルボー接続**（参照 fig-7 の文法）で結ぶ。接続契約: (a) スタブ幅＝子のインデント幅とし、線と箱は必ず接触する、(b) スパイン（縦罫）は**最初の子の中心から最後の子の中心まで**だけを結ぶ（`first-child` は上半分、`last-child` は下半分を描かない）、(c) 1 子のときはスパインを描かず直線で結ぶ、(d) leaf 層も同一パターンの再帰。全幅スラブを廃止し各ノードは内容幅。線の途切れ・はみ出しは構造的に発生しない実装（モックの `.lt-child::before/::after` 方式）を正とする。
8. **waterfall@2（renderer-SVG）** — 参照完全再現: Y 軸（矢印付き）＋目盛＋単位ラベル、バー間の点線コネクタ、期首/期末=紺塗り、増加=白抜き紺枠、減少=橙（絶対値最大の減少因子のみ塗り、他はアウトライン）、負値 ▲ 表記、値はバー内、因子名はバー上/下、X ラベルは期首/期末の下。幾何は既定の Decimal＋`displayPrecision` 規約で算出し整数座標へ丸める。`RENDERER_SVG_ALLOWLIST` は `{slope@2, waterfall@2}` へ更新（slope@1 は @2 昇格に伴い除去）し、waterfall の viewBox は `0 0 640 360` 完全一致。バー数上限は 7（期首・期末含む）。現行の `bars | columns` 二形態はこの単一 SVG 形式へ統合し廃止する。
9. **slope@2** — 既定=紺、注目系列のみ teal。端点ドット＋**線の終端に数値と系列名を結合表示**（中央浮遊ラベル廃止）。viewBox `0 0 600 220` は維持。
10. **evidence-map@2** — 紺結論箱＋左スパイン（2px 罫）＋枝で根拠カード（アウトライン）へ物理接続。浮遊線分を廃止。線種=確度（実線/破線/点線）と確度テキスト併記は維持。
11. **bars@2（昇格）** — 図ヘッダ＋行: ラベル / 紺バー / 右端に値（nowrap）。主題の 1 本だけ teal。最大 10 行。
12. **kpi@2（昇格）** — 紺 5px リング円（8.5rem）＋内部に大数字（2.1rem/700・紺）＋単位小書き＋円下キャプション。1 行 3 個まで、最大 5 個。

## アーキテクチャ影響

- **バージョン**: 12 形式すべて `@2`。`component-vocabulary.json`・`component-ir.schema.json`・`assembly.schema.json`・`registry.json`（digest は `shasum -a 256` の厳密値）・`TRUSTED_RENDERERS`・checker 期待・bad フィクスチャを同一タスクで原子的に更新する。
- **bars/kpi 昇格**: 拡張ゲート 10 手順に完全準拠（relationshipKind: bars=`quantitative-comparison`、kpi=`headline-metrics` を語彙へ追加）。
- **checker 追加規則**: (a) 数値ラベル・tier ラベルの nowrap/最小幅検査、(b) 文中強調の 1 説明文 1 個上限、(c) ハイライト 1 図 1 箇所上限、(d) 確度語彙 3 語固定、(e) waterfall@2 の SVG 二重ゲート（slope 方式）、(f) 定量図の `unitLabel` 必須。
- **廃止**: CSS 版 waterfall（`ve-wf-start-*`/`ve-wf-len-*` の整数%量子化機構ごと）。互換節経由の旧 waterfall マークアップは従来どおり legacy 扱いで残る（canonical とは区別される）。
- **fixtures**: 既存フィクスチャ・examples を新スタイルで再生成する。旧骨格（72rem main）の fixture-claude-*/fixture-pi-* は新骨格で再生成する。
- **script は引き続きゼロ**（static-first 維持。双方向部品は P2 の別 spec）。

## ダークモード方針

参照スライドは紙面（ライト）前提のため、ダークは**同一色相の明度再マッピング**で追随する（トークン表の dark 列）。紺面は明度を上げ、白抜き文字は `--dg-on-primary`、アウトライン系は `--dg-line` の明色化で対応。文字色トークンを面の塗りに流用する現行の欠陥はこのリスタイルで消滅する。

## 検証

1. 四層検査（既存）＋上記 checker 追加規則の bad フィクスチャで失敗を固定。
2. 12 形式のギャラリーフィクスチャをビルドし、Playwright（`--channel chrome`）でライト/ダーク × デスクトップ/モバイル（390px）の 4 系統スクリーンショットを取得、レビュー時の判定材料とする。判定基準: (a) 数値・ラベルの折返しゼロ、(b) 見出し・本文への図形要素の重なりゼロ、(c) ハイライト・強調の上限遵守、(d) 接続線の途切れ・はみ出しゼロ（logic-tree のエルボー、chevron のループレール、evidence-map のスパイン、waterfall のコネクタ）。
3. モックとの目視突合（モックが見た目の正）。

## 範囲外（別 spec）

- P1: アンカー目次・モバイル matrix 縦積み縮退
- P2: copy-as-prompt・選択 ask・line@1・data: 画像解禁
- バッジ頻度 lint（列単位確度宣言の IR 追加を含むため、認知負荷系の別 spec に回す）

## リスクと弱い前提

- 12 形式×拡張ゲート 10 手順の一括改訂は変更面積が大きく、レジストリ digest・checker 期待・スキーマ enum のドリフトが起きやすい。原子的コミットとドリフト検査で抑えるが、実装は形式ごとに独立検証可能なタスクへ分割する（plan で詳細化）。
- 紺パレットの WCAG コントラストは light で検証済みの組合せのみ（紺×白 8.7:1）。淡紺セル×濃文字、dark の各組合せは実装時にコントラスト検査を通す。
- clip-path（chevron）の印刷互換はブラウザ依存が残る（Chromium/Safari で確認、それ以外は best effort）。
