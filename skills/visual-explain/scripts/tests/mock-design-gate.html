ビジュアル基準 v1 デザインゲート

この文書は目視レビュー専用のモックであり、checker には掛けない。代表4画面で「色＝判断状態のみ」「左端整列」「型階層」を目で確認するためのものである。

代表4画面を承認できれば、以降の描画実装は見た目の判断を蒸し返さずに進められる。

visual-explain のビジュアル基準 v1 を、モノクロ構造＋判断3色・4段階の型階層・8pxグリッドで確定する。

あなたが決めること: この4画面の見た目を「正」として承認するか、指摘を戻して再提示させるか。

  • light / dark の両テーマで、色が付くのは判断状態（選択・推奨・注意）だけであること。
  • 広幅1280pxと狭幅390pxの双方で、見出し・本文・図・キャプションの左端が一直線に揃うこと。

見た目案を一覧で並べると、判断3色を残しつつ構造をモノクロ化する案が最も認知負荷が低い。

検討した見た目案 認知負荷 実装コスト 判定
現行の多色UIを維持 高（色の意味が曖昧） 小 却下
モノクロ構造＋判断3色 v1 低 中 採用
完全モノクロ（判断色なし） 中（状態が読めない） 小 却下

8pxグリッド＋4段型階層 低 中 併用採用

承認フローは縦の背骨で読め、軽微な改訂だけがレールで承認へ直行する。

決定・依頼・仮説の3種の ask は同じ枠を共有し、色が付くのは決定の選択チップと既定案バッジだけである。
